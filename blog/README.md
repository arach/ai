# Blog

This directory already holds the longer Talkie-style writeups:

- `part1-*.md` through `part6-*.md`
- architecture notes and post drafts

For day-by-day work logs, use `blog/daily/`.

## Two lanes

- `blog/part*.md` for polished narrative posts
- `blog/daily/*.md` for TIL-style notes, benchmark runs, and small progress logs

## Daily post format

Use a simple filename and keep the post skimmable:

- `YYYY-MM-DD-short-slug.md`

Suggested sections:

- `What changed`
- `What I learned`
- `Results`
- `Next`

Keep these posts lightweight. The goal is to preserve the workstream and the
reasoning behind a run, not to turn every daily note into a full article.
